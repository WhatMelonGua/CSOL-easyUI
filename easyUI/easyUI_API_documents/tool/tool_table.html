工具表 tool_table

提供了对Lua语言中 table类型 处理的一些常用函数

Functions

.deepCopy (table) [·] 完全复制目标表table [深拷贝]
.deepNil (table) [·] 完全清空 目标表table，触发 Lua的 GC垃圾回收
.surfaceNil (table) [·] 对 目标表table 仅进行第一层清空
若内含子表，仅会断开键值对关联，子表不会被删除
.deepPrint (table) [·] 层层输出 目标表table， 进入新层以 '->' 表示
.allCount (table) [·] 对 目标表table 进行表层元素的计数 (包括key和索引都将被计数入内)
._deepNil (table) [·] 完全清空 目标表table，不触发 Lua的 GC垃圾回收

Functions

.deepCopy (table)
涉及表的 深拷贝 和 浅拷贝。

浅拷贝

当您用 a_Table = b_Table 直接赋值新表时，只是把 a 指向了 b
任意更改 a / b 的元素 都会对双方造成影响(可以理解为链接关系)

深拷贝

使用deepCopy完全生成一个一模一样的新表，与原表互不影响。

Parameters:

  • table table Lua中的表类型 数据
.deepNil (table)
仅使用 table = nil，若表中套有子表 如:table[1] 也是个table
则table[1]实则是没被清空的，只是断开了 table[1]的链接关系。
使用deepNil将 表中表 也完全清空删除！

Parameters:

  • table table Lua中的表类型 数据
.surfaceNil (table)
清空表的第一层内容，若有表中表 则不清空。

Parameters:

  • table table Lua中的表类型 数据
.deepPrint (table)
深挖表数据，打印表中表，而不是只输出一个 table。

Parameters:

  • table table Lua中的表类型 数据

Usage:

table= {inner={1,2},{3,4},5}
tool_table.deepPrint(table)
以上示例的执行结果（CSOL中按下 '~'按键 打开控制台）如下: 
1 -> 1: 3
1 -> 2: 4
2: 5
inner -> 1: 1
inner -> 2: 2
.allCount (table)
返回表的 元素个数

如表 table = {1, 2, 3, key=4}
#table = 3, 而 tool_table.allCount(table) = 4

Parameters:

  • table table Lua中的表类型 数据

Return:

  • number number 表内元素的 个数
._deepNil (table)
仅使用 table = nil，若表中套有子表 如:table[1] 也是个table
则table[1]实则是没被清空的，只是断开了 table[1]的链接关系。
使用deepNil将 表中表 也完全清空删除！
此举不进行垃圾回收。
虽然垃圾回收期间占用内存，但对于UI.Box类 向Lua外部申请的数据，是有必要用GC及时回收的。

Parameters:

  • table table Lua中的表类型 数据
