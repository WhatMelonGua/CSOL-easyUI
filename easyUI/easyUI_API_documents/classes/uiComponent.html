伪类 class_uiComponent

此伪类 用于针对指定的UI.Box及其sets生成UI组件[Box组]，并提供了一系列常用的组件的变换操作。

其提供的pos、scale等属性允许您打组Box完成组件化后，对组件进行自由移动、缩放、克隆复制等。

相信这将对您 UI界面 的 多组件管理 提供极大便利。

名词解释：

—— GC：Lua语言的垃圾回收操作，会自动清空不使用的变量等释放内存

—— (global) ：指 同种伪类 共用一个此属性(不同伪类间不共用)，是全局属性


示例:

uiComponent伪类支持 连续操作。（请先了解大致 Functions功能 后再查看此技法）
只要该函数在文档内未标识Return返回值，均可连续进行操作！
--如 :positionSet() 未标注返回值则 支持 连续操作 并返回当前组件对象(return self)；
--而 :boxCount() 有返回值则不支持，后方不可以加:Function继续操作

如下，我们一口气注册setArg、构建UI.Box并设置位置、缩放进行绘制： 
uiBox = class_uiComponent.new():setRegister(
        {x=0,y=0,width=40,height=40,r=0,g=0,b=0,a=255},
        {x=5,y=5,width=30,height=30,r=255,g=255,b=255,a=255}
        ):boxSupply():positionSet({x=300,y=300}):scaleSet(2):Draw()
其效果和下方代码相同，写起来却更省事，当然您如果不习惯这种写法也可以使用下方： 
uiBox = class_uiComponent.new()
uiBox:setRegister(
{x=0,y=0,width=40,height=40,r=0,g=0,b=0,a=255},
{x=5,y=5,width=30,height=30,r=255,g=255,b=255,a=255}
)
uiBox:boxSupply()
uiBox:positionSet({x=300,y=300})
uiBox:scaleSet(2)
uiBox:Draw()

Attributes

ui_type (global) 标识了此伪类的属别，可能仅在debug时使用。
其值为'uiComponent' (string) 不可更改
size_design (global) 当游玩者的电脑屏幕尺寸与您的设计尺寸不同时，会导致UI.Box界面不一致，此属性为UI.Box的尺寸提供屏幕修正。
当开启屏幕修正时，uiComponent将自动根据尺寸差别将您的box进行映射，保证所有内容的可见性。(会导致轻微的拉伸，程度据屏幕差异大小而变)
其默认值为 {width=1920,height=1080} (table) 可更改
fix_factor (global) 屏幕尺寸的修正比例，其储存的是 玩家尺寸/设计尺寸将在Lua启动运行瞬间完成初始化。 [ 不需要您进行人工管理 ]
fix_factor = {x=?,y=?} (table) 不可更改
fixAble_default (global) 创建组件时，默认是否进行 屏幕修正
（优点:所有内容都可以绘制，缺点：对应不同屏幕会有可能有拉伸） (boolen)
boxs 组件的box表，储存了组件绘制所需的所有UI.Box (table)
sets 组件的set表，储存了组件 boxs 表内所有UI.Box依次对应的setArg (table)
set输入类型同官方setArg: {x=?,y=?,width=?,height=?,r=?,g=?,b=?,a=?}
pos 组件的绘制位置
pos={x=?,y=?} (table)
scale 组件的缩放大小
scale.x代表横向缩放大小，scale.y则代表纵向缩放大小
scale = {x=?,y=?} (table)
plugins 组件绘制时，针对单个box和set进行中间处理的 Function表
其只接收 function(函数)类 的元素插入表内。
plugins = {function, function, ....} (table)
callback (uiComp) 组件绘制完成要执行的 回调函数，为空/非函数 则不执行。 (function)

Functions

为方便标识，class_uiComponent下称 uiComp [self指uiComp实例本身]

:sizeSet (size) 设置您的 设计尺寸 (全局设置，您只需在开始指定一次即可)
.new (fixAble) [·] 创建新的 实例化uiComponent对象
.fromTemplate (template, fixAble) [·] 通过 模板 创建新的 实例化uiComponent对象
:boxCreate (num) 为uiComp创建 num 个UI.Box用于绘制
:boxDelete (deleteNum) 从 self.boxs 中删除 deleteNum 个已存在的UI.Box 触发GC
:boxClear ( ) 清空该组件中已存在的所有 UI.Box [清空self.boxs] 触发GC
:boxCount ( ) 返回该组件已使用的UI.Box数量
:boxSupply ( ) 根据组件 self.sets 中setArg数量，自动补充创建对应数量的UI.Box
:boxAdapt ( ) 将组件使用的UI.Box数裁补至 self.sets 的对应数量 (多删少补)
:setRegister ( ...\setArg\ ) 接收setArg形式的可变长参数，向组件注入逐个UI.Box的set信息
:setCopy (obj) 将obj对象的sets表复制给该组件 [obj可为 uiComp / set表 ]
:setLink (obj) 将该组件的sets表链接至obj对象 [obj可为 uiComp / set表 ]
当obj的setArg更改时，该组件也会受到影响
:setClear ( ) 清空该组件的 sets表，所有setArg将被删除 [清空self.sets]
:setRotate (ro, turn) 通过改变sets 将组件以 顺时针[默认]/逆时针 旋转90° ro 次
[使用请查看详情，存在必要的补充说明]
:setMirror (axis) 改变sets，将该组件以 X/Y (axis) 为对称轴 作对称
:positionSet (position) 设定该组件在屏幕上的 绘制位置
:scaleSet (scale) 设定该组件绘制的 缩放比例
:localMove (move) 移动该组件的 中心点 默认为{x=0,y=0}即左上角
:localCenter ( ) 将组件的中心点 移动至组件的中心
:Draw (package) 绘制该组件。
package参数可选[不填]，为plugin插件函数的接收值
:Show ( ) 将该组件设定为可见 默认即为可见
:Hide ( ) 设置该组件为隐藏不可见
:merge (component,front) 将该组件与另外一 uiComp 合并，并返回合并后的新组件
:mergeInit (component) 将该组件与另外一 uiComp 合并，并初始化，并返回合并后的新组件
:clone ( ) 克隆/复制该组件，返回一个新的uiComp对象
:linkCopy ( ) 链接复制该组件，返回一个新的uiComp对象
新对象除Box独立，sets将与该组件关联变化，如:setLink( self )的效果
:destroy ( ) 彻底清除/销毁该组件 触发GC
:callbackSet (function) 传入一个函数，作为每次绘制完毕的回调函数 (绘制完毕后立刻执行)
:callbackClear () 清除该组件的回调函数
:pluginRegister ( .../function/ ) 向组件注册 plugin绘制中间件 函数，见 plugin函数 详解
:getArea ( ) 返回该组件的占地尺寸，{左上角，右下角}的坐标
:_setNormalize (set, factor) uiComp的内置函数，不供给用户使用，用于对setArg进行pos，scale，屏幕修正的计算

Parameters

setArg 官方UI.Box的 set 结构参数
vector 二维向量结构参数 可表示位置、缩放方向等
size 表征尺寸宽高的结构参数
color 颜色结构参数
plugin Function (package) plugin绘制插件函数
callback Function (uiComp) uiComponent回调函数


Attributes

ui_type
标示着该对象在 easyUI 的类型为uiComponent

String:

  • String "uiComponent"
size_design
编写Lua脚本 电脑的 屏幕尺寸，称为 设计尺寸； 玩家的实际屏幕尺寸称为 游玩尺寸
其存在意义在于 一套UI界面能在不同的屏幕上得以完全展示[注意：不是完美展示]

Table:

  • width number 设计尺寸的 宽度
  • height number 设计尺寸的 高度
fix_factor
easyUI 对组件的修正系数，导入Lua脚本至游戏的一瞬间脚本将自动完成计算，无需您进行设置

Table:

  • x number 其值 = 客户端.width / 设计端.width
  • y number 其值 = 客户端.height / 设计端.height
fixAble_default
使用 class_uiComponent.new() 时省略参数，默认的传递值。
注意：使用 :sizeSet() 函数，此值将自动变为 true，也就是接受屏幕修正

Boolean:

  • fixAble_default number 默认为 false（即不进行屏幕修正），若您使用了 sizeSet则会自动变为true。
boxs
储存组件绘制所需UI.Box的表，可以通过索引获取对应的UI.Box
如uiComp.boxs[1] 获取第一个UI.Box

Table:

  • i UI.Box i 代表 索引数字
sets
储存组件绘制所需setArg的表，可以通过索引获取对应的setArg
如uiComp.sets[1] 获取第一个setArg。
绘制时 uiComp.boxs[i] 的Box 将以 uiComp.sets[i] 的setArg 对应索引进行设定
若 Box数 > setArg数，多余的Box设为透明

Table:

  • i setArg i 代表 索引数字
pos
标识组件中心点的绘制位置，Draw函数执行时，将在此处展开绘制，基本值为{x=0,y=0}

Table:

  • x number 横坐标
  • y number 纵坐标
scale
缩放系数，基本值为{x=1,y=1}，将以组件中心点展开缩放

Table:

  • x number 横向的缩放系数
  • y number 纵向的缩放系数
plugins
储存 plugin绘制中间件(函数)，其编写方法见 plugin 函数
根据数字索引进行储存、取用
多个plugin函数存在时， 会将上一个 plugin处理后的内容 传给下一个层层处理得到最终结果
若中间某个plugin无返回值，则以 上个plugin的返回值(还没有则上上个 直到原始值) 进行继续传递

Table:

  • i function i 代表 索引数字
callback (uiComp)
回调函数，其详细介绍见 callback函数 参数模板

Function:

  • callback function 回调函数

Functions

:sizeSet (size)
设置设计屏幕的尺寸大小
注意：您应该避免使用 UI.ScreenSize ()进行赋值，而应该是有明确值的{width=?, height=?}的表
此函数使用后，将自动把 fixAble_default 属性 变为 true
之后创建的新组件 默认都接收屏幕修正。（若您想某些不接受，则使用 .new(false) ）

Parameters:

  • size size Table，包含width和height的宽高尺寸属性
.new (fixAble)
创建一个uiComp实例，返回一个初始化的uiComp

Parameters:

  • 可选 fixAble bool 不填默认为true，是否对该组件应用屏幕修正 true为应用
.fromTemplate (template, fixAble)
通过现有模板创建uiComp实例，您也可以利用 UI Box Painter 构建自己的模板库供他人使用！

Parameters:

  • template function 模板函数，避免模板的setArg对内存造成占用，以函数封装
  • 可选 fixAble bool 不填默认为true，是否对该组件应用屏幕修正 true为应用
:boxCreate (num)
为uiComp的boxs表中追加创建 num 个 UI.Box
若原来存在UI.Box，将不会被清空，而是在原有基础上增加 num 个

Parameters:

  • num number 需要创建的UI.Box数量
:boxDelete (deleteNum)
将从该组件的self.boxs表中由后向前删除 deleteNum 个UI.Box
一般当sets表的 setArg 发生数量变更时用来释放UI.Box的占用

Parameters:

  • deleteNum number 要向前删除的UI.Box数量
:boxClear ()
清空组件的 boxs表，所有UI.Box将被清除并进行垃圾回收。
:boxCount ()
对组件消耗/利用的 UI.Box总量 进行计数

Return:

  • value number 该组件消耗的 UI.Box数量
:boxSupply ()
若现有boxs表的 UI.Box数量不足
则根据组件现有的sets表内 setArg总数 与现有UI.Box之差 向boxs表 补充对应数量的 UI.Box
注意：本方法不会将多余的UI.Box删除，若需要请使用 boxAdapt()
:boxAdapt ()
根据现有sets表内的 setArg数量，将boxs表内的 UI.Box 通过 增加/删除 使之与其数量对应
若需要删除 UI.Box 则触发GC 垃圾回收机制。
:setRegister (...)
向组件注册 setArg， 其传入参数可变长。（即您可以单次注册 1个 / 多个 setArg）
注册后的setArg将 追加至 uiComp的 sets表内， 不会对原有的setArg产生影响

Parameters:

  • setArgs setArg 传入标准的setArg (单个/多个)
:setCopy (obj)
将传入 obj 的sets 列表 复制给调用本方法的组件
此方法应与 setLink 区分；setCopy是 复制了一份新的sets表给至 该组件的 sets
后续双方的sets发生改动后 互不影响

Parameters:

  • obj uiComponent | table 传入参数可为 uiComp对象 | setArg的表(如uiComp.sets表)
:setLink (obj)
将传入 obj 的sets 列表 链接给调用本方法的组件
此方法应与 setCopy 区分；setLink是 双方共用此sets表(obj与该uiComp)
只要一方对sets做出更改，其余通过Link链接的uiComp/obj 都会受到影响
当您希望构建一群 显示效果完全相同的组件时，使用setLink共用一组sets表可以减少内存开销。

Parameters:

  • obj uiComponent | table 传入参数可为 uiComp对象 | setArg的表(如uiComp.sets表)
:setClear ()
清空调用该方法的组件的sets表，表内所有 setArg 将被清除
:setRotate (ro, turn)
调用该方法来实现 组件的图形学旋转，每次旋转90°，默认按照顺时针方向旋转。

事实证明，当您把组件缩小非常小(有时scale设置为 0.5甚至以下，旋转会出错)
推荐您试一下，如果您的电脑上没问题，那就是没问题的！
这并不是easyUI算法的问题，而是lua的 程序接口设计问题，无法弥补！
解决方法：对组件关闭屏幕修正 or 保持较大的scale（>=1）
沿 组件中心点 进行旋转。

Parameters:

  • ro number 进行旋转的次数，如 ro=2 代表旋转180°
  • turn bool 默认为 true 代表顺时针旋转，false代表逆时针。
:setMirror (axis)
将该组件 以 axis 轴进行对称。
例如'Y'为左右对称，'X'为上下对称。
对称轴即 沿组件中心点进行伸展的x/y轴。

Parameters:

  • axis string 以 此轴 进行轴对称反转，值为 'X' | 'Y' 或 'x' | 'y'
:positionSet (position)
设置组件的绘制位置。

Parameters:

  • position vector position.x:绘制的横坐标 | position.y 绘制的纵坐标
:scaleSet (scale)
设置组件的绘制缩放比例。

Parameters:

  • scale vector | number 缩放比例，若参数scale 类型为 number 则自动转换为 {x=scale,y=scale}
:localMove (move)
对组件的 中心点 进行偏移。
中心点 可以通过几何中心 来进行类比说明：
(如一个边长c的正方形 其几何中心在 {x=c/2,y=c/2}的位置，而非{x=0,y=0})

Parameters:

  • move vector 在原有基础上位移的距离。
:localCenter ()
将组件 中心点 进行自动居中。
其方法是 根据组件UI.Box的 左上角、右下角坐标 ； 取两坐标之间平分点为中心点。
:Draw (package)
绘制该组件。
接收参数package可不填写(nil)
package将作为参数传入共 plugin Function 使用， 其应用方法见 plugin Function 介绍。

Parameters:

  • package T T 表示任何可能的数据类型。一般您可能并不会用到package参数，忽略不填即可。
:Show ()
将 组件 全面显示；默认情况下 组件的 UI.Box都是可见的。
其原理是对所有 boxs表 的 UI.Box执行 UI.Box:Show()操作。
:Hide ()
将 组件 全面隐藏。
其原理是对所有 boxs表 的 UI.Box执行 UI.Box:Hide()操作。
:merge (component,front)
将该组件与 传入的 component(uiComp对象) 合并为一个组件。
合并时，会考虑到二者的 位置 & 缩放 信息 基于self组件 进行合并！
合并后的组件，位置 和 缩放 继承self组件，传入的uiComp的setArg会由程序执行变换映射。
返回初始化组件的 中心点 与self相同

merge并不会销毁进行组合的 2个组件，若您需要可以手动调用:destroy进行销毁。
同样的，merge返回的 合并组件 不储存本身的合并历史，即您无法通过合并得到的组件重新分解得到原来的组件
是不可逆的合并

Parameter:

  • component uiComponent 需要组合的uiComponent对象。
  • 可选 front bool merge主体 是否覆盖在 需组合的uiComp对象 上方。
    默认为 false，即若传入的 uiComp和 self组件 有重合部分，uiComp将遮挡住self

Return:

  • mergeComponent uiComponent 返回 二组件 组合后的 uiComponent对象。
:mergeInit (component)
将该组件与 传入的 component(uiComp对象) 合并为一个组件。
默认 传入参数 遮盖 self组件(类比 merge 函数 )
[如果文字描述 2个merge函数 并不清晰，作者建议实际实践下，对不同的合并进行移动/缩放 看差异]
合并时，会考虑到二者的 位置 & 缩放 信息 基于屏幕 进行合并！
这意味着 若您放大2倍 进行合并后，返回的 合并组件 将以原来大小的 2倍 作为基础大小。
返回初始化组件的 中心点 在左上角

merge并不会销毁进行组合的 2个组件，若您需要可以手动调用:destroy进行销毁。
同样的，merge返回的 合并组件 不储存本身的合并历史，即您无法通过合并得到的组件重新分解得到原来的组件
是不可逆的合并

Parameter:

  • component uiComponent 需要组合的uiComponent对象。

Return:

  • mergeComponent uiComponent 返回 二组件 组合后的 uiComponent对象。
:clone ()
将该组件 自我克隆一份 返回克隆所得的 全新uiComponent对象
应与 linkCopy() 进行区分，clone所得的 新对象 与 该组件本身 完全独立，互不影响！

Return:

  • cloneComponent uiComponent 返回 全新独立的uiComponent对象，您对其进行任何操作将与原组件无关了。
:linkCopy ()
将该组件的 sets复用，返回新的uiComponent对象。
linkCopy() 所得的 新对象 与 原组件 共用一个sets表，为链接关系。
当您改变任一对象的sets表内容时，另外的组件也会受到影响，并在绘制时体现出来。

因此推荐您 一般使用clone()方法 防止误操作导致存在链接关系的UI组件互相影响，导致显示混乱
当然，您能 不再更改/合理更改 sets表 时，可以放心大胆的使用 linkCopy()，将为客户端 节省一部分 内存空间，加快您的代码执行速度！

Return:

  • copyComponent uiComponent 返回 与原组件的 sets 存在链接关系的 新uiComponent对象
:destroy ()
销毁 该组件对象本身。
销毁后，其向UI申请创建的 UI.Box将全部回收，为其他绘制提供空间。
同时请注意，如有其他组件的 sets 与其存在链接关系 (linkCopy | setLink 操作) ，其他组件将丢失sets 无法绘制
:callbackSet (function)
设定 该组件绘制完毕后的 回调函数。
当该组件的Draw()方法被调用后，回调函数将会立刻执行一次。
绘制多次则 执行多次，可重复执行。
若您只需要执行一次，可在 回调函数中 对该组件执行 uiComp:callbackClear() 消除回调函数

Parameters:

  • function callback Function 回调函数，点击左方蓝色 callback Function字样进行 了解及使用。
:callbackClear ()
清除该组件的回调函数，原理是 self.callback = nil，为其置空即可。
:pluginRegister (...)
向 该组件注册 plugin 插件函数。
plugin Function将在对 boxs表 的每个 UI.Box进行 :Set(set)操作时 进入处理。
此时 plugin函数 为您提供了一个 更改set/box 属性的通道，在原有值上进行一系列操作后再进行Set。
plugin对set的改变将 不会影响 sets表 内的SetArg！可放心使用。 了解使用 plugin

Parameters:

  • plugin plugin Function 类比setRegister()函数，其传入参数也是可变长的：
    意味着您可以传入 一个/多个 plugin函数，以逗号分隔
:getArea ()
获取该组件的占地坐标
其将返回 一个Table，包含了组件的 左上角坐标 & 右下角坐标 信息，以vector类型表示。

Return:

  • area Table area[1]为左上角坐标，area[2]为右下角坐标。二者以 vector 类型进行表示。
:_setNormalize (set,factor)
该方法对 组件内的所有 setArg进行了修正，实现了 组件的位移、缩放、屏幕尺寸修正适应
其内部只是简单的对 长方形setArg进行的一系列数学计算后，获取应当设定的最终 setArg
uiComp的绘制流程:

new新建实例对象 -> 注册setArgs -> boxSupply创建UI.Box -> 位置、缩放等属性设定 -> Draw
-> plugin处理(若有) -> _setNormalize最终修正 -> 回调函数触发(若有)

Parameters:

  • set setArg
  • factor self.fix_factor

Return:

  • set setArg

Usage:

以下plugin 通过 pluginRegister(...) 注册后，将把对应组件绘制为纯红色
而不同box的 透明度 将由 Draw(package) 函数 传入的 表package的 a属性 的值指定 
function pluginExample ( box, set, i, package )
    -- 传入参数的顺序为此 : box为当前绘制的UI.Box | set为当前box的初始setArg |
    --i为box和set在表内的索引 | package是Draw绘制时您可手工增加的参数，不传则为nil
    box = box
    -- 一般不会对box 进行处理，可省略
    set.r = 255
    set.g = 0
    set.b = 0
    -- 将所有box的setArg的r值都提至255最高，变为纯红色，当然这是您可以自定的
    set.a = package.a
    -- package即:Draw(package)函数传入的 package函数，所有plugin共用，可以用key值区分不同plugin的参数
    -- 一系列更复杂的setArg更改操作，未更改的将继承原值
    -- 您或许可通过此指定不同x,y坐标的box对应不同色彩 | 传入时间t，按照更改setArg
    return box, set
    -- 是的，一系列操作完后，请务必在最后返回 box和set，否则此plugin对set的更改将不生效
    -- 当然，即时不return如此，其他的操作是有效的，如果您愿意，可以在这里操作一些setArg外的其他内容
end

Parameters:

  • box UI.Box 正在进行绘制的UI.Box对象
  • set setArg 此时UI.Box对应的setArg
  • i number UI.Box | setArg 在 boxs表 | sets表 对应的 索引数字
  • package T 可以是任意类型
    当 plugin插件 > 1，推荐作为表传入，通过key值设定 不同plugin可能用到的参数
    package从 uiComp:Draw(package) 函数传入。

Return:

  • box UI.Box 当前Box，当然您也可以指定成为其他的UI.Box，不过并不建议
    这会让程序十分复杂。
  • set setArg 更改后的SetArg
